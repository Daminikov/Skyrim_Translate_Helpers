# Skyrim_Translate_Helpers

Скрипты Python для более удобного и автоматизированного перевода модификаций с помощью нейросетей (Grok, ChatGPT, Claude и.т.д).

---

```text

│
├── build_database.py                    # Скрипт 1: Извлечение базы из ESP-ESM Translator
├── generate_prompts_to_AI_TXT.py        # Скрипт 2: Генерация промптов для .txt файлов
├── generate_prompts_to_AI_XML.py        # Скрипт 3: Работа с .xml файлами
│
├── base/                          # Папка с базами переводов

│   └── _compiled_glossary.pkl     # Кеш оптимизированного словаря
```

# 🛠 Скрипт 1: build_database.py
🎯 Назначение
Извлекает переводы из базы ESP-ESM Translator 4.35 и сохраняет их в удобные JSON-файлы в папке base/. Предварительно необходимо конвертировать базу данных из формата .eet в .xml.

## ⚙️ Настройки (в начале скрипта)
```
EET_DATABASE_PATH = r"S:\ESP-ESM Translator 4.35\BDD"  # Путь к базе EET
BASE_FOLDER = Path('base')                             # Куда сохранять JSON

# Разрешенные группы (GRUP). Берем то, что будет в базе:
ALLOWED_GRUPS = [...]
```
## ⚠️ Важно: Запускайте этот скрипт только при необходимости:

При первом запуске.
После обновления базы ESP-ESM Translator.
После добавления новых модов в базу.

# 📝 Скрипт 2: generate_prompts_to_AI_TXT.py
Генерирует готовые промпты для нейросетей на основе .txt файлов. 
Автоматически извлекает релевантный глоссарий из базы для каждого конкретного файла.

## 📋 Приоритеты словарей
При загрузке базы скрипт учитывает следующие приоритеты (от низшего к высшему):

Приоритет 1 — Все обычные JSON-файлы.
Приоритет 2 — Custom_Pet9948.json.
Приоритет 3 — BDD_SkyrimSE_EN-RU.json.
Приоритет 4 — User_Base.json (перезаписывает всё!).

## 🚫 Исключения
Слова из файла Exceptions.json не попадают в итоговый глоссарий промпта.

## 📄 Шаблоны промптов
Шаблоны лежат в prompts/templates/. Первая строка в файле шаблона — это его описание (она игнорируется при генерации промпта).

💡 Использование
В папке prompts/ появится файл {имя_файла}_PROMPT.txt.

Откройте сгенерированный промпт.
Скопируйте блок SYSTEM PROMPT в нейросеть.
Скопируйте блок USER CONTENT вторым сообщением.
Получите идеальный перевод с сохранением формата КЛЮЧ | Текст.

# 🔄 Скрипт 3: generate_prompts_to_AI_XML.py
Обрабатывает файлы переводов.xml из ESP-ESM Translator. Извлекает тексты из тегов <ORIGINAL>, генерирует промпт для перевода, а затем позволяет вставить переводы обратно в XML через теги <TRADUIT>.

Примечание: Этот скрипт использует только шаблоны, начинающиеся на XML (например, XML_skyrim.txt). Остальные игнорируются.

## ⚙️ Процесс работы
Шаг 1: Извлечение оригиналов
Скрипт читает XML файл и находит все теги <ORIGINAL>:
```
<ORIGINAL>Thanks for downloading Completionist</ORIGINAL>
<TRADUIT />
```
Шаг 2: Генерация промпта
Для каждого найденного оригинала формируется строка с ее порядковым номером. Затем извлекается глоссарий и формируется промпт (как в Скрипте 2).
```
String_number_56 | Thanks for downloading Completionist
String_number_57 | Another text here
```
Шаг 3: Создание файла для перевода
В папке input/ создается пустой файл {имя}_translated.txt. Вставьте туда полученный от нейросети переведенный текст.

Шаг 4: Вставка переводов обратно
После того как вы заполнили файл перевода, скрипт спросит:
```
Загрузить перевод в XML? (да/нет):
```
Если ответ "да", скрипт:
Читает файл из input/.
Извлекает номер строки и перевод.
Находит соответствующий пустой тег <TRADUIT> в исходном XML.
Вставляет перевод: <TRADUIT>Ваш перевод</TRADUIT>.

# 💡 Полезные советы
User_Base.json — ваш главный инструмент: Если нейросеть переводит что-то неправильно или не в том контексте, добавьте правильный перевод в этот файл. Он имеет высший приоритет и перезапишет любые совпадения из других баз.

Exceptions.json — чистый глоссарий: Если какие-то слова засоряют промпт и не требуют глоссария (например, "Note", "Valid Items"), добавьте их сюда. Они не попадут в промпт, но нейросеть всё равно переведет их правильно в контексте.

Шаблоны промптов: Создавайте разные шаблоны для разных задач:

skyrim.txt — стандартный перевод Skyrim.

XML_skyrim.txt — для XML-файлов (игнорируется в txt-скрипте).

custom.txt — ваш экспериментальный шаблон для особых случаев.
